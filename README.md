# nets-and-firewalls
Тема: Настройка сетевого фильтра UFW для сервера компании
Предмет: Администрирование / Сетевые технологии
Сценарий: Сервер компании «Ядер-Кола»

Исходные данные для примера
Сервер компании

IP-адрес: 192.168.55.50

ОС: Linux (Ubuntu)

Firewall: ufw

ПК администратора

IP-адрес: 192.168.55.90

Развёрнутые сервисы
Сервис	Порт	Протокол
SSH	22	TCP
Prometheus	9090	TCP
Grafana	3000	TCP
Node Exporter	9100	TCP

Доступ по SSH разрешён только с ПК администратора

Доступ к Prometheus и Node Exporter разрешён:

с ПК администратора;

с локального хоста (localhost)

Доступ к Grafana разрешён:

аналитикам (192.168.55.10–192.168.55.30);

разработчикам (192.168.55.91–192.168.55.128)

Все остальные входящие соединения должны быть заблокированы

Все исходящие соединения должны быть разрешены


Для реализации базовой защиты были заданы политики:

запрет всех входящих соединений;

разрешение всех исходящих соединений.

Это гарантирует, что доступ возможен только по явно разрешённым правилам.

Для повышения безопасности доступ по SSH ограничен одним доверенным IP-адресом — ПК администратора.

Сервисы мониторинга являются внутренними и не должны быть доступны посторонним узлам.
Поэтому доступ разрешён только:

администратору;

локальной системе.

Диапазон IP-адресов разработчиков (192.168.55.91–192.168.55.128) не может быть корректно описан одной CIDR-подсетью, поэтому он был разбит на несколько подсетей, что позволяет:

строго соответствовать заданному диапазону;

исключить предоставление избыточного доступа.

Использован принцип least privilege

Исключён несанкционированный доступ к сервисам

Все требования задания выполнены в полном объёме

Конфигурация корректна с точки зрения сетевой адресации

Проверка конфигурации

Для проверки применяется команда:

sudo ufw status verbose

Ожидаемый результат:

входящие соединения запрещены по умолчанию;

разрешены только необходимые правила;

исходящие соединения разрешены.
